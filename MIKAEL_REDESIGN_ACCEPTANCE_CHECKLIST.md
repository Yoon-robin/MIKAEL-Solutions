# MIKAEL 리디자인 검수 기준

이 문서는 Claude Code가 리디자인 후 스스로 확인해야 할 완료 기준입니다.

## 1. 브랜드/언어 기준

- [ ] 사용자 노출 브랜드는 `MIKAEL Solutions`로 유지된다.
- [ ] 사용자 노출 UI에서 `OSIRIS`, `Osiris` 문구가 되살아나지 않는다.
- [ ] 주요 UI 문구는 한국어 기준으로 자연스럽다.
- [ ] 기술 약어는 필요 시 유지한다: OSINT, CVE, DNS, WHOIS, SSL/TLS, API.
- [ ] 한글 문구에 무리한 `uppercase` 스타일을 적용하지 않는다.

## 2. 기능 보존 기준

- [ ] 지도 렌더링이 유지된다.
- [ ] 레이어 토글 기능이 유지된다.
- [ ] 지역 프리셋 이동이 유지된다.
- [ ] 정보 피드/실시간 경보가 유지된다.
- [ ] 시장/우주기상 패널이 유지된다.
- [ ] 정찰 도구함의 각 기능 탭이 유지된다.
- [ ] CCTV/라이브 피드 뷰어가 유지된다.
- [ ] 공유 링크 기능이 유지된다.
- [ ] API 라우트 동작을 불필요하게 변경하지 않는다.

## 3. 레이아웃/UX 기준

- [ ] 메인 지도는 화면의 중심으로 남아 있다.
- [ ] 좌우 패널이 지도를 과도하게 가리지 않는다.
- [ ] 패널 헤더, 상태값, 숫자 데이터의 시각 계층이 명확하다.
- [ ] 모바일 하단 탭/드로어가 터치하기 쉽다.
- [ ] 작은 화면에서 텍스트가 잘리거나 겹치지 않는다.
- [ ] 경보색은 중요한 상태에만 사용한다.
- [ ] 금색/시안/녹색/빨강 계열이 일관된 의미를 가진다.

## 4. 성능/안정성 기준

- [ ] 새 대형 의존성을 추가하지 않는다. 추가가 필요하면 이유를 명확히 남긴다.
- [ ] 지도 마커/애니메이션을 과도하게 늘리지 않는다.
- [ ] Docker 이미지 빌드가 가능해야 한다.
- [ ] 저사양 서버 환경을 고려해 불필요한 무거운 효과를 피한다.

## 5. 검증 명령

필수:

```bash
npm run build
```

권장:

```bash
npm run lint
```

주의: `npm run lint`는 기존 코드베이스 전반의 `any` 타입/unused 경고로 실패할 수 있습니다. 리디자인으로 새로 만든 오류인지 기존 오류인지 구분해서 보고하세요.

## 6. 배포 전 확인

운영 반영 전에는 다음을 확인하세요.

```bash
docker build -t osiris:local .
docker run --rm -p 127.0.0.1:3001:3000 \
  -e NODE_ENV=production -e PORT=3000 -e HOSTNAME=0.0.0.0 \
  osiris:local
```

별도 터미널에서:

```bash
curl -sS --max-time 10 http://127.0.0.1:3001/api/health
curl -sS -I --max-time 10 http://127.0.0.1:3001 | sed -n '1,20p'
```
